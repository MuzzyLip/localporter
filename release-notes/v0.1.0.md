# LocalPorter v0.1.0

LocalPorter `v0.1.0` is the first usable release. The goal of this release is to ship a usable native desktop baseline for local port inspection, menu bar / tray interaction, and packaging validation.

## Highlights

- Native Rust desktop app built with `egui`/`eframe`.
- macOS supports both standalone window mode and menu bar panel mode.
- Local port collection currently focuses on:
  - TCP `LISTEN`
  - UDP bound ports
  - optional TCP "show all" expansion in the UI
- Each row can surface best-effort process metadata:
  - process name
  - parent launcher name
  - uptime
  - CPU usage
  - memory usage
  - PID
  - command
- Basic actions are available for opening local services in the browser and terminating killable processes.
- Runtime logs are written to the app configuration directory under `logs/`, with one log file per launch instance.

## Release Artifacts

### macOS

- `LocalPorter-0.1.0-macos-universal.dmg`
- Universal app bundle inside the DMG, supporting both:
  - Apple Silicon (`aarch64`)
  - Intel (`x86_64`)

Notes:
- This build is currently unsigned and not notarized.
- Users may need to manually bypass Gatekeeper on first launch.

### Windows

- `LocalPorter-0.1.0-windows-x64.zip`
- `LocalPorter-0.1.0-windows-x64.msi`

Notes:
- The ZIP package is a portable distribution.
- The MSI package is intended for normal install and upgrade flows.

## Known Limitations

- Process and port visibility is limited to what the current OS permissions allow.
- Some fields may fall back to `Unknown` when the system does not expose them reliably.
- The first public release should be treated as an early usable version, not a long-term stable contract.

## Upgrade Guidance

- macOS: replace the existing app in `/Applications`.
- Windows MSI: install the newer MSI to upgrade the existing installation.
- Windows ZIP: extract and replace manually.

## Logs for Issue Reports

When reporting issues, attach the newest log file from the app configuration directory's `logs/` folder. Filenames use sortable UTC timestamps, for example:

`localporter-20260708-021530-123Z.log`
